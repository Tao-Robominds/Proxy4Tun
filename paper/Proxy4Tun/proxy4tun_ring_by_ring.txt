Intrinsics4Tun: a proxy-based reflective framework for label-free tunnel point-cloud segmentation at deployment
Highlights
* Label-free point cloud tunnel-lining segmentation at deployment.
* Ring-level projection-based pipeline employing boundary detection.
* Gravity-referenced tunnel bottom alignment is essential.
* Offline BO-calibrated intrinsic proxy serving as an mIoU indicator.
* Self-refinement increasing mean mIoU 3.46-fold over static SAM4Tun.


Abstraction
Automated segmentation of tunnel-lining components from point clouds enables segment-wise inspection of joint gaps, leakage, and deformation. Deployment typically lacks ground-truth labels, yet the system must still adapt and decide whether to accept, refine, or flag each result. We present Intrinsics4Tun, a deployment-time label-free quality control framework for a projection-based, ring-level tunnel-lining segmentation pipeline. Offline, BO is used as an experience generator on a small labelled calibration panel to identify sensitive parameters and intrinsic diagnostics that correlate with mIoU; these are then frozen as an intrinsic proxy for deployment-time self-assessment and refinement. We further expose a structural failure mode of ring-level projection: unresolved rotational ambiguity can produce physically misaligned segmentations, which we address with a gravity-referenced bottom-stabilisation prior that anchors each ring before projection. During deployment, LLM-proposed/reflection-based adaptations search bounded candidate layouts using the calibrated proxy. On 50 held-out rings from five tunnels, mean mIoU improves from 0.137 for the static SAM4Tun baseline to 0.475 after intrinsic refinement, a 3.46-fold increase. The rule-based seeded Intrinsics4Tun pipeline reaches 0.167 (+21.5%), bottom stabilisation reaches 0.290 (+73.9% over seeded), and proxy-guided refinement provides the final lift to 0.475 (+63.7% over stabilised). These results show that compact intrinsic diagnostics can support label-free adaptation and quality control for auditable tunnel-lining segmentation.
1. Introduction
Gap
At deployment, without labelled data, segmentation systems need to perform both adaptation and quality control (whether to accept, refine, or flag).


* Engineer-designed systems are auditable but less adaptable (manually tuning).
* Deep-learning systems are adaptable with a labelled dataset but less auditable.


Intrinsics4Tun is label-free at deployment and the proxy is constructed by observable internal qualities.
Intrinsics4Tun
Pipeline  
Figure 1: Intrinsics4Tun converts each ring point cloud into a stabilised 2D depth map, detects candidate segment boundaries, selects the best segment layout using an intrinsic proxy, and reprojects the selected segmentation to 3D.


* The point cloud is processed at ring level by projecting the tunnel surface into a 2D depth map through unfolding, denoising, and enhancement. 
* Domain-informed structural priors are used to initialise parameters, including tunnel geometry, expected segment count and cyclic order. 
* Boundary detection is then performed on the depth map, using oblique-line evidence for the keystone region and near-horizontal boundary evidence for the remaining segment interfaces. 
* Candidate segment layouts are generated by varying K position, segment offsets, and related detection parameters via LLM-proposed/reflection-based adaptations, but they are accepted only when intrinsic structural metrics improve.  
* The iterative process continues within a fixed budget and terminates when intrinsic improvement plateaus or the budget is exhausted.
* The selected 2D segmentation is reprojected back to the 3D point cloud; mIoU is used only for offline evaluation.


Intrinsic proxy
  

Figure 2: Design-time construction of the intrinsic proxy. BO is used on a small labelled design panel as an experience generator to expose parameter sensitivities, failure cases, and intrinsic signals that correlate with mIoU. The calibrated proxy and structural guardrails are then frozen for label-free deployment.
* It uses a small selected labelled design dataset covering diverse tunnel and point-cloud conditions to calibrate the relationship between intrinsic quality measures and segmentation mIoU.
* At design time, BO is used to identify sensitive parameters and construct an intrinsic proxy that acts as an approximate mIoU indicator.
* At deployment-time, the system remains label-free.
   * It initialises the pipeline using domain-informed structural priors, including tunnel geometry, expected segment count, cyclic segment order, and gravity-referenced bottom alignment;
   * Candidate adaptations, including LLM-proposed/reflection-based variants, are evaluated using the intrinsic proxy and structural guardrails;
   * A candidate is accepted only when it improves the intrinsic objective and does not fall below the stabilised baseline safety floor; otherwise the system abstains to the baseline.
   * Iteration terminates when intrinsic gains saturate or the adaptation budget is exhausted.

Contributions:
1. We formulate label-free deployment quality control for tunnel-lining point-cloud segmentation using intrinsic, observable pipeline metrics. 
2. We use design-time Bayesian Optimization (BO) on a small labelled calibration set to identify intrinsic metrics and parameter sensitivities that correlate with segmentation quality. 
3. We identify rotational ambiguity in ring-level 3D-to-2D unfolding as a structural failure mode, and introduce gravity-referenced tunnel-bottom alignment to stabilise the projection. 
4. We construct an intrinsic proxy for estimating segmentation quality, enabling accept/refine/abstain decisions without deployment labels. 
5. We integrate these components into an auditable end-to-end framework, improving mean mIoU by approximately 3.46-fold over the static baseline.

3. Methodology
Task definition
Given a segmental tunnel-lining ring point cloud, the task is to identify the keystone segment (K), its two adjacent segments, and the remaining base segments.
* Each ring contains exactly one K segment.
* The angular position of K may vary across rings.
* The total number of lining segments is typically 6 or 7.
* Segment labels follow a fixed cyclic order, for example K, B1, A1, A2, A3, A4, B2, up to rotation
* Non-K segment offsets are expected to remain relatively consistent within the same tunnel type.
  

Figure 4: Example ring-level boundary detection for the segmental lining task. The detected horizontal interfaces define the ordered base segments, while the keystone region is identified from the distinctive central/oblique line evidence. The goal is to iteratively recover the single K segment, its two adjacent segments, and the remaining cyclically ordered base segments.
Dataset
* BO calibration panel: 
   * 6 labelled K-bearing, covering density, coverage, K-span, and joint-orientation variations.
* Held-out evaluation panel: 
   * 50 random rings, 10 each from five tunnels, including two regular staggered (T1-2), one continuous (T3), and two irregular staggered (T4-5).

BO
BO is used as a design-time experience generator, not as the final optimisation engine. On the labelled calibration panel, BO explores parameter changes and failure cases to reveal which intrinsic, observable signals track mIoU/OA. These signals are then distilled into a frozen proxy and guardrails for label-free deployment.

* struct_missing_ids 
   * What it is: How many expected segment IDs were missing before completion/repair.
   * Intuition: More missing IDs means the structure is less complete, so quality should drop.
   * Coefficient sign: Negative (-0.023682) -> more missing pieces lowers predicted mIoU.
* depth_row_nonempty_ratio 
   * What it is: The fraction of vertical rows in the depth image that contain at least one valid depth value, computed and recorded by the pipeline’s depth-quality audit file.
   * Intuition: Higher audited coverage usually means cleaner depth input and easier segmentation.
   * Coefficient sign: Positive (+0.201192) -> strongest term in this formula.
* geom_boundary_gap 
   * What it is: The coefficient of variation (std/mean) of consecutive boundary spacings around the ring, derived from candidate boundary positions (after wrap-around).
   * Intuition: It measures how uneven the candidate’s boundary layout is; this captures candidate-specific geometric structure that helps distinguish competing candidates within the same ring.
   * Coefficient sign: Positive (+0.081910) -> higher boundary-gap variability is associated with higher predicted mIoU.
Overall performance
Overall, Intrinsics4Tun increases mean mIoU from 0.1374 for the static SAM4Tun baseline to 0.4750 after refinement, a total lift of +0.3376 and about 3.46x relative improvement. The largest gains come from stabilisation (+0.1233) and intrinsic refinement (+0.1849), showing that both tunnel-bottom stabilisation and proxy-guided adaptation contribute materially to final performance.

5. Discussion
Key findings
The results suggest three main findings.  
1. Ring-level segmentation is an effective formulation for tunnel-lining point clouds. By decomposing the tunnel into structurally meaningful rings, the segmentation task becomes more constrained and easier to regularise using geometric priors such as segment count, cyclic order, and keystone position.  
2. A well-calibrated intrinsic proxy can provide effective label-free quality control. Although the proxy is calibrated using labelled data at design time, deployment does not require ground-truth labels. The system can evaluate candidate segmentations using observable structural signals and select improved outputs without directly computing mIoU.
3. Intrinsic self-refinement enables practical adaptation. Instead of relying on a fixed parameter setting, Intrinsics4Tun can search over candidate adaptations and accept those that improve the intrinsic objective. This allows the pipeline to adapt to variations in tunnel diameter, segment count, point density, and ring geometry while remaining auditable.


Limitations
The intrinsic proxy is not perfectly accurate. It approximates segmentation quality using a small number of observable indicators, and therefore can miss some ring-level failure cases or produce regressions when the proxy score and true mIoU are not fully aligned (Table 6 in Appendix). In this study, the proxy is deliberately kept compact, using fewer than five main indicators, which improves interpretability but limits expressiveness. 


The intrinsic search is also not exhaustive. The current refinement process evaluates a bounded set of candidate adaptations, so finer-grained parameter exploration may identify additional improvements. Future work could improve search efficiency by learning from accumulated self-refinement experience, for example through a reinforcement-learning (RL) policy, a retrieval-based knowledge base, or a learned proposal model that prioritises promising candidate adaptations.
6. Conclusions
This paper presents Intrinsics4Tun: a compact intrinsic proxy for deployment-time, label-free quality control in tunnel-lining point-cloud segmentation. The proxy is calibrated at design time using a small labelled panel and BO-generated experience, then fixed for deployment. At runtime, it supports accept/refine/abstain decisions using observable internal signals, without ground-truth labels.  Results on the held-out panel show that proxy-guided refinement improves mIoU over static and pre-refinement stages. Gravity-referenced stabilisation is an important prerequisite because it reduces projection inconsistency and makes candidate scoring more comparable across rings.  The current proxy is intentionally compact and interpretable, but this also limits representation capacity in difficult cases. Future work will focus on improving proxy calibration and ranking reliability, adding uncertainty-aware abstention, and improving candidate proposal efficiency while keeping runtime decisions auditable.
