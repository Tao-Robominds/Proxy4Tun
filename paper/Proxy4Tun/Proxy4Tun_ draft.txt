Proxy4Tun: self-refine framework for label-free tunnel point-cloud segmentation at deployment
Highlights
* Label-free point cloud tunnel-lining segmentation at deployment.
* Ring-level projection-based pipeline employing boundary detection.
* Gravity-referenced tunnel bottom alignment is essential.
* Offline BO-calibrated intrinsic proxy serving as an mIoU indicator.
* Self-refinement increasing mean mIoU 3.46-fold over static SAM4Tun.


Abstraction
1. Introduction
Gap: Ring-level motivation
  

Figure 1. Motivation for ring-level segmentation and proxy tuning. Even within the same tunnel, rings can exhibit markedly different point-cloud characteristics. For example, ring T5-1/R273 is much sparser than T5-1/R270, producing larger white/background regions. Treating the tunnel as a single homogeneous object would therefore hide ring-specific variation and lead to suboptimal segmentation parameters. Our pipeline segments the tunnel at the ring level, unfolds each ring independently into a depth image, applies a proxy tuner to adapt the segmentation parameters for that ring, and then reprojects the optimised segmentation back onto the 3D point cloud.


________________
Pipeline


  

Figure 2. Ring-level segmentation pipeline with proxy-guided self-refinement. Each ring-level point cloud is first unfolded into a 2D depth-map projection. During the line-detection stage, a proxy tuner iteratively adjusts the detection parameters using a proxy score with guardrails, continuing until the score reaches a plateau or the tuning budget is exhausted. The detected boundaries then define the regions assigned as 2D segments, and the resulting segmentation is finally reprojected back onto the original 3D point cloud.
________________
BO-calibrated proxy
  

Figure 3. Learning an observable proxy objective from Bayesian optimisation (BO). BO  is first used on selected GT-available rings to explore how different detection parameters change the unfolded depth map, boundary layout, and segmentation output. From these BO trials, we identify observable features that correlate with segmentation quality, including depth completeness, boundary regularity, and segment balance. These features are then used to train a proxy equation that predicts expected mIoU without requiring ground-truth labels at deployment time, enabling the tuner to select better parameters for new rings using only intrinsic visual/geometric cues.

________________


Proxy tuner
  

Figure 4. Proxy-guided LLM tuner at deployment. Given a new ring-level depth map. The LLM proposes boundary detection  parameters based on the point cloud characteristics. Each parameter set is evaluated by the learned proxy score, which estimates expected mIoU without ground-truth labels. The tuner iteratively refines the parameters until the proxy score reaches a plateau or the evaluation budget is exhausted, and the highest-scoring candidate is selected as the final segmentation estimate.

Data preprocessing
  Figure 5. Resolving rotational ambiguity by bottom alignment. The ring unfolding process has an inherent rotational phase ambiguity: the same 3D ring can be projected with different angular starting points, causing the observed tunnel surface to be split across the depth-map boundary and leaving large white/empty regions in the 2D image. We therefore apply bottom alignment, shifting the unfolding so the physical tunnel bottom is consistently placed at the same reference row. This removes the ambiguous rotation axis, producing a more continuous depth map.
BO
BO is used as a design-time experience generator, not as the final optimisation engine. On the labelled calibration panel, BO explores parameter changes and failure cases to reveal which intrinsic, observable signals track mIoU/OA. These signals are then distilled into a frozen proxy and guardrails for label-free deployment.


Proxy
Employ the Ridge model for standard offline supervised, GT-based mIoU training on BO-derived data. During deployment, runtime scoring is exclusively performed using the three non-GT observable features.

5. Discussion
Key findings
The results suggest three main findings.  
1. Ring-level segmentation is an effective formulation for tunnel-lining point clouds.
2. A well-calibrated intrinsic proxy can provide effective label-free quality control. 
3. Intrinsic self-refinement enables practical adaptation. 
Limitations
1. The intrinsic proxy is not perfectly accurate. (Table 6 in Appendix)
2. The intrinsic search is also not exhaustive.
